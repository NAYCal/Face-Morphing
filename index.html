Overview

This program will be divided up into parts, with the final product creating a 'morph' animation from one face to another. "A morph is a simultaneous warp of the image shape and a cross-dissolve of the image colors. The cross-dissolve is the easy part; controlling and doing the warp is the hard part. The warp is controlled by defining a correspondence between the two pictures. The correspondence should map eyes to eyes, mouth to mouth, chin to chin, ears to ears, etc., to get the smoothest transformations possible". - CS 180 Fall 2023

Part 2: Computing the "Mid-way Face"

In this part, we will compute the "Mid-way Face" between the first image to the final image. First, we will find the transitional image between the first and final image by taking a weighted average. Then, we will find the affine transformation matrix from both images to the transitional image. Finally, we will use the new found affine transformation matrix to perform wrapping and color dissolution. To find the affine transformation function, we can use the corresponding points taken during Part 1 to solve for the affine transformation matrix. Since there are 6 unknowns, we will need 3 points (Note: Each point has an x and y component). After finding the affine transformation matrix, we can use inverse wrapping to take pixels from our first image into our new "Mid-way Face" image.

Part 5. Caricatures: Extrapolating from the mean

Finally, I will extrapolate from the mean to create a caricature of myself. In the context of face shapes, this meant using the average face shape (vector v) as a baseline to emphasize unique features in a specific face (vector w). The differential vector (w - v) encapsulates the distinctive attributes of vector w compared to the mean. To accentuate these unique traits, we add a scaled version of this differential vector back to w, using α(w−v) where α is a positive scalar. This process not only highlights but also exaggerates the unique features of face w, effectively creating a caricature by intensifying the deviations from the mean face shape. The scalar α adjusts the level of exaggeration applied to the unique facial features, providing a means to control the caricature's degree of distortion from the original.

Computing the PCA basis

To compute the PCA basis or eigenfaces of a dataset, we start by taking the mean image and then subtract it from each image, resulting in a set of centered images. While one could calculate the covariance matrix following this step, we opted for a more efficient approach by employing a PCA library, saving time and optimizing the process. This library, presumably using Singular Value Decomposition (SVD), facilitates the computation of eigenvalues and eigenvectors (the eigenfaces), crucial components that highlight the main variations in the dataset. We then select the top k eigenfaces associated with the highest eigenvalues, capturing the significant facial features and effectively reducing data dimensionality. Consequently, each image can be represented as a combination of these selected eigenfaces, ensuring efficient data compression and preservation of vital facial features for subsequent analysis.
